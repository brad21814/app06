# Data Model: reCAPTCHA Verification

## Verification Request
Sent from the client to the server-side API routes.

```typescript
interface RecaptchaRequest {
  recaptchaToken: string; // The token generated by the client-side reCAPTCHA v3 script
}
```

## Verification Result (Google API)
The response from `https://www.google.com/recaptcha/api/siteverify`.

```typescript
interface GoogleRecaptchaResponse {
  success: boolean;       // whether this token was a valid token for your site
  score: number;         // the score for this request (0.0 - 1.0)
  action: string;        // the action name for this request (important for verification)
  challenge_ts: string;  // timestamp of the challenge load (ISO format yyyy-MM-dd'T'HH:mm:ssZZ)
  hostname: string;      // the hostname of the site where the reCAPTCHA was solved
  'error-codes'?: string[]; // optional error codes
}
```

## Internal Verification Outcome
The result of the `verifyRecaptcha` utility in `lib/auth/recaptcha.ts`.

```typescript
interface RecaptchaOutcome {
  success: boolean;       // True if score >= threshold and success is true
  score: number;         // The actual score returned by Google
  error?: string;        // Human-readable error message if verification fails
}
```
